# Задача 2. Pizza Toppings (Project Euler 281)

## Условие

Пицца (круг) разрезана на `m × n` одинаковых кусочков.

- Есть `m` различных видов начинок (`m ≥ 2`)
- Каждую начинку нужно положить ровно на `n` кусочков (`n ≥ 1`)

**Определение**  
`f(m, n)` = количество способов распределить начинки по кусочкам, если:

- Отражения считаются различными
- Повороты — не считаются различными

Примеры:

- `f(2, 1) = 1`
- `f(2, 2) = 2`
- `f(3, 1) = 2`
- `f(3, 2) = 16`

## Задача

Найти сумму всех `f(m, n) ≤ 10¹⁵`.



## Решение на C# 

Используется лемма Бёрнсайда.  
Для каждого `(m, n)` перебираются `k` от 0 до `m·n - 1`, вычисляется `g = gcd(m·n, k)`, проверяются условия и суммируются вклады.  
Сложность — приемлемая для `f(m, n) ≤ 10¹⁵`.


## Решение на Prolog 

Реализованы предикаты:

- `gcd(A, B, G)` — наибольший общий делитель
- `factorial(N, F)` — факториал
- `f(M, N, Value)` — вычисление `f(m, n)`
- `sum_all_f(Limit, Sum)` — сумма всех `f(m, n) ≤ Limit`

Используется рекурсивный перебор `k` с отсечением.  
На малых тестах работает корректно, полный расчёт до `10¹⁵` требует длительного времени.




## Решение на F# 

- Нет циклов, только хвостовая рекурсия
- Нет переменных (immutable)
- Используется `bigint` для предотвращения переполнения
- Функции: `gcd`, `factorial`, `f`, `collectForM`, `collectAll`, `sumAllF`
