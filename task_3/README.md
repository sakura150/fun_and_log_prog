# Описание задачи: (см. https://projecteuler.net/problem=139)

Необходимо найти количество пифагоровых троек, которые соответствуют таким треугольникам, 

поместив четверку таких одинаковых треугольников в квадрат со стороной, равной их гипотенузе, 

мы можем получить квадрат, внутри которого 

есть незаполненный треугольниками участок - другой квадрат, причем сторона этого квадрата такова, что 

он может поместиться натуральное число раз в построенный треугольниками квадрат и заполнить внешний квадрат полностью, 

периметр искомых треугольников не должен превышать 100_000_000.

# Решение

# Шаг 1 - выразим условие через катеты a и b

Переформулируем задачу:

Сторона квадрата, то есть, c в Пифагоровой троке (a,b,c) должна делиться на сторону квадрата, 

получаемого при построении квадрата из 4х прямоугольных треугольников - обозначим последнее как n

Выразим n: n = sqrt(c^2 - 4*S), где S - площадь прямоугольного треугольника 

Выразим c и S через a и b: n = sqrt(a^2+b^2-2ab) => a-b

Таким образом, требования можно переписать так: sqrt(a^2+b^2) делится нацело на (a-b), причем a+b+(a^2+b^2) <= 100_000_000


# Шаг 2 - оптимизация решения

На текущем этапе есть математиеская формулировка задачи, но алгоритм, который сейчас 

заключается в том, чтобы проверять каждую пару натуральных значений (a,b) на удовлетворение описанным 

условиям, очень не эффективный.

Воспользуемся понятием примитивной Пифагоровой тройки (a,b,c), 

которое заключается в том, что c^2 = a^2+b^2, и НОД(a,b,c)=1.

Существует теорема, говорящая о том, что такие тройки могут быть порождены как: 

a = m^2 - n^2, b = 2mn, c = m^2 + n^2, для каждого m>n>0, таких что НОД(m,n)=1.

Любая другая Пифагорова тройка (непримитивная) может быть порождена как:

a = k*(m^2 - n^2), b = k*(2mn), c = k*(m^2 + n^2), для каждого m>n>0, таких что НОД(m,n)=1.

Таким образом, можно рассматривать значительно меньшее число троек - большая часть 

из них (которые не являются примитивными) может быть получена путем подбора k, такого что:

k*(m^2 - n^2) + k*(2mn) + k*(m^2 + n^2) <= 100_000_000, без операции проверки на делимость, поскольку, 

очевидно, коэффициент на делимость не влияет (он сокращается).

# Описание решения задачи на F#
1. Генерация примитивных троек реалиуется функцией isPrimitiveValid m n

Проверяет пары (m, n) на соответствие условиям:
(m-n) нечётное
GCD(m,n) = 1
c делится на |a-b| без остатка

2. Подсчёт всех кратных треугольников

Вместо генерации каждого треугольника отдельно, решение использует оптимизацию:

Для каждого примитивного треугольника с периметром p₀

Количество допустимых кратных = maxPerimeter / p₀

Это позволяет избежать создания списков и экономит память.

3. Агрегация результата

Суммируется количество треугольников для всех валидных пар (m, n).
Циклы не используются 

Вместо условных операторов if используется сопоставление с образцом

Все данные неизменяемы — используются let вместо mutable переменных.

Функции высшего порядка
* List.collect — генерация всех пар (m, n)
* List.map — преобразование элементов
* List.sumBy — агрегация результатов с функцией проекции

Рекурсия

gcd реализован через хвостовую рекурсию

generateScaledTriangles (в полной версии) использует рекурсию вместо циклов


# Описание решения на Prolog

Предикат valid_mn(M, N) отвечает за генерацию параметров N и M. 

Перебираются все пары натуральных чисел M и N, где:
M > N > 0
M и N взаимно просты (gcd(M, N) = 1)
M и N имеют разную четность ((M - N) mod 2 = 1)
Это условия формулы Евклида для генерации примитивных пифагоровых троек.

Предикат prim_triple(MaxM, A, B, C) отвечает за генерацию примитивных троек

Для каждой валидной пары (M, N) вычисляются стороны треугольника по формуле Евклида:
a = M² - N²
b = 2·M·N
c = M² + N²
Это дает примитивную пифагорову тройку (не имеющую общего делителя).

Проверка главного условия задачи осуществляется предикатом tiling_prim(MaxM, A, B, C)

Проверяется ключевое условие задачи:
Вычисляется разность катетов: Diff = |a - b|
Проверяется делимость: c mod Diff = 0

Предикат count_mult(A0, B0, C0, MaxP, K, Count) отвечает за генерацию всех кратных троек 

Каждая примитивная тройка (a₀, b₀, c₀) порождает бесконечное семейство троек:
(k·a₀, k·b₀, k·c₀), где k = 1, 2, 3, ...
Рекурсивно подсчитываются все такие тройки, у которых:
Периметр P = k·(a₀ + b₀ + c₀) ≤ MaxPerimeter

Наконец, предикат count_triangles(MaxPerimeter, Count) подсчитывает общее количество троек 

Вычисляется верхняя граница для M:

MaxM = ⌊√(MaxPerimeter/2)⌋ + 1

Для каждой примитивной тройки, удовлетворяющей условию замощения:

Находится количество допустимых кратных

Все количества суммируются с помощью:

findall/3 - собирает все подсчеты в список

sum_list/2 - суммирует элементы списка
