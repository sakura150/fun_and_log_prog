# Задача 1. Square Progressive Numbers

## Условие

Положительное целое число `n` делится на `d`. Частное `q`, остаток `r`.  
Известно, что `d, q, r` — это три последовательных члена **геометрической прогрессии** (не обязательно в порядке `d, q, r`).

Пример:  
`58 / 6 = 9` (остаток 4).  
`4, 6, 9` — последовательные члены геометрической прогрессии со знаменателем `3/2`.

Такие `n` называются *progressive numbers*.

Некоторые из них являются полными квадратами.  
Известно: сумма всех progressive perfect squares ниже `100 000` равна `124657`.

### Задача

Найти сумму всех progressive perfect squares ниже `10¹²`.

## Алгоритм (математический вывод)

1. Пусть три последовательных члена геометрической прогрессии:  
   `a, a·t, a·t²`, где `t = p/s` в несократимом виде, `p > s ≥ 1`, `gcd(p,s)=1`.

2. Чтобы все три числа были целыми, `a` должно делиться на `s²`.  
   Положим `a = k·s²`.

3. Тогда три числа:  
   `k·s²,  k·s·p,  k·p²`.

4. Перебираем все перестановки этих чисел в `(d, q, r)` и проверяем:
   - `n = d·q + r`
   - `r < d`
   - `n` — полный квадрат

5. Подходят только две перестановки:

   **Формула 1:**  
   `d = k·p²,  q = k·s·p,  r = k·s²`  
   `n = k²·s·p³ + k·s²`

   **Формула 2:**  
   `d = k·p²,  q = k·s²,  r = k·s·p`  
   `n = k²·s²·p² + k·s·p`

6. Перебираем `p, s, k` с ограничениями:
   - `p > s ≥ 1`, `gcd(p,s)=1`
   - `k ≥ 1`
   - `n < 10¹²`
   - Проверяем, является ли `n` полным квадратом

7. Собираем уникальные `n` в множество, суммируем.


## Оценка сложности

- Количество итераций ~ `10⁴` (перебор `p, s, k`)
- Проверка на квадрат — O(1)
- Итоговая сложность — пренебрежимо мала, программа выполняется за доли секунды


## Используемые технологии

- C# (.NET 8)
- HashSet для уникальности
- Целочисленная арифметика (long)

## Решение на C# (императивный стиль)

Используется математический вывод:

Три последовательных члена геометрической прогрессии:  
`a, a·t, a·t²`, где `t = p/s` в несократимом виде, `p > s ≥ 1`.

Положим `a = k·s²`, тогда три числа:  
`k·s², k·s·p, k·p²`.

Подходят две перестановки:

**Формула 1:**  
`n = k²·s·p³ + k·s²`

**Формула 2:**  
`n = k²·s²·p² + k·s·p`

Перебираем `p, s, k` с условиями:
- `gcd(p, s) = 1`
- `n < 10¹²`
- `n` — полный квадрат

Суммируем уникальные `n`.


## Решение на Prolog (логический стиль)

Реализованы предикаты:

- `gcd(A, B, G)` — наибольший общий делитель
- `is_square(N)` — проверка, является ли число полным квадратом
- `generate_n(Limit, N)` — генерация прогрессивных квадратов
- `sum_progressive_squares(Limit, Sum)` — суммирование

Используется рекурсивный перебор `p, s, k` с отсечением.  
На малых тестах работает корректно, полный расчёт до `10¹²` требует длительного времени.


## Решение на F# (функциональный стиль)

- Нет циклов, только хвостовая рекурсия
- Нет переменных (immutable)
- Функции: `gcd`, `isSquare`, `generateFromFormula1`, `generateFromFormula2`, `generateAll`, `sumProgressiveSquares`
